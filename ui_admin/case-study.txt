\# Quản lý Bán Hàng Thời Trang --- ## 1. Tên Dự Án ### Fashion E-Commerce Website Management System Hệ thống Website Bán Hàng Thời Trang — nền tảng thương mại điện tử tích hợp cho phép quản lý toàn bộ hoạt động bán hàng thời trang trực tuyến, từ quản lý sản phẩm, đơn hàng đến trải nghiệm mua sắm của khách hàng. --- ## 2. Công Nghệ Sử Dụng | Thành phần | Công nghệ | |---|---| | Frontend | ReactJS (Hooks, React Router, Axios) | | Backend | Java Spring Boot | | Database | MySQL 8.0 | | ORM | Spring Data JPA (Hibernate) | | Bảo mật | Spring Security + JWT (JSON Web Token) | | Email | Spring Mail | | Giảm code boilerplate | Lombok | | Build Tool | Maven | | IDE | IntelliJ IDEA Ultimate / Community | --- ## 3. Bối Cảnh Dự Án Ngành thời trang đang chứng kiến sự chuyển dịch mạnh mẽ sang kênh bán hàng trực tuyến. Các cửa hàng và thương hiệu thời trang cần một hệ thống website bán hàng hiện đại để: - Quản lý danh mục sản phẩm đa dạng với nhiều thuộc tính (màu sắc, kích thước, chất liệu) - Xử lý đơn hàng nhanh chóng, chính xác và minh bạch - Cung cấp trải nghiệm mua sắm mượt mà, thân thiện với người dùng - Quản lý tồn kho, theo dõi doanh thu theo thời gian thực - Tăng tỷ lệ chuyển đổi thông qua các tính năng khuyến mãi, voucher - Xây dựng lòng trung thành khách hàng qua hệ thống đánh giá và tài khoản cá nhân --- ## 4. Mục Tiêu Dự Án ### 4.1 Mục Tiêu Chức Năng - Xây dựng hệ thống web bán hàng thời trang đầy đủ chức năng cho cả khách hàng và quản trị viên - Cho phép khách hàng duyệt sản phẩm, đặt hàng, theo dõi đơn hàng và thanh toán trực tuyến - Cung cấp công cụ quản lý sản phẩm, đơn hàng, người dùng, khuyến mãi cho Admin - Tích hợp xác thực bảo mật bằng JWT và phân quyền rõ ràng theo vai trò - Gửi email thông báo tự động khi đặt hàng, xác nhận tài khoản ### 4.2 Mục Tiêu Đo Lường (KPI) | KPI | Giá trị | |---|---| | Thời gian tải trang | < 2 giây | | Số lượng người dùng đồng thời hỗ trợ | ≥ 100 | | Thời gian xử lý đơn hàng | < 5 phút | | Độ chính xác tồn kho | 99.9% | | Unit Test coverage | ≥ 70% | --- ## 5. Phân Quyền Hệ Thống ### Cấu Trúc Phân Cấp \`\`\` 👑 Admin — Toàn hệ thống 🙍 Customer — Mua sắm cá nhân 🌐 Guest (Khách vãng lai) — Xem sản phẩm, chưa đăng nhập \`\`\` ### Admin \*\*Phạm vi:\*\* Toàn hệ thống - Quản lý danh mục sản phẩm (thêm / sửa / xoá / tìm kiếm / phân cấp cha-con) - Quản lý sản phẩm và biến thể (màu sắc, kích thước, số lượng tồn kho) - Quản lý đơn hàng (xem, cập nhật trạng thái, huỷ) - Quản lý yêu cầu trả hàng (Xem chi tiết, duyệt/từ chối yêu cầu, cập nhật trạng thái hoàn tiền) - Quản lý tài khoản khách hàng (khoá / mở khoá) - Quản lý khuyến mãi và voucher - Xem báo cáo doanh thu, thống kê sản phẩm bán chạy - Thực hiện bán hàng trực tiếp tại quầy, tra cứu nhanh tồn kho và giá bán. ### Customer \*\*Phạm vi:\*\* Cá nhân - Đăng ký / đăng nhập / quản lý hồ sơ cá nhân - Duyệt và tìm kiếm sản phẩm theo danh mục, lọc theo giá / màu sắc / kích thước - Thêm sản phẩm vào giỏ hàng (Cart) và danh sách yêu thích (Wishlist) - Đặt hàng, chọn địa chỉ giao hàng, áp dụng voucher - Tạo yêu cầu trả hàng cho đơn hàng đã hoàn thành và theo dõi tiến trình xử lý. - Theo dõi trạng thái đơn hàng và lịch sử mua hàng - Đánh giá và nhận xét sản phẩm đã mua ### Guest (Khách vãng lai) \*\*Phạm vi:\*\* Chỉ đọc - Xem danh sách và chi tiết sản phẩm - Tìm kiếm sản phẩm - Đăng ký / đăng nhập để tiếp tục mua hàng > \*\*Lưu ý thiết kế DB:\*\* Bảng \`users\` cần cột \`role\` (ENUM: ADMIN / CUSTOMER) và cột \`is\_active\` để hỗ trợ khoá tài khoản. JWT token lưu thông tin role để phân quyền API. --- ## 6. Yêu Cầu Chức Năng ### 6.1 Chức Năng Trong Phạm Vi (In Scope) #### A. Chức Năng Khách Hàng (Customer) - \*\*Xác thực tài khoản:\*\* Đăng ký, đăng nhập, đăng xuất, quên mật khẩu (gửi email reset), xác nhận email - \*\*Hồ sơ cá nhân:\*\* Xem và cập nhật thông tin cá nhân, đổi mật khẩu, quản lý địa chỉ giao hàng - \*\*Duyệt sản phẩm:\*\* Xem danh sách sản phẩm, lọc theo danh mục / giá / màu sắc / kích thước, tìm kiếm theo tên - \*\*Chi tiết sản phẩm:\*\* Xem ảnh, mô tả, giá, đánh giá, chọn biến thể (màu / size) - \*\*Giỏ hàng (Cart):\*\* Thêm / xoá / cập nhật số lượng sản phẩm - \*\*Yêu thích (Wishlist):\*\* Thêm / xoá sản phẩm yêu thích - \*\*Đặt hàng:\*\* Chọn địa chỉ, phương thức thanh toán, áp dụng voucher, xác nhận đơn - \*\*Trả hàng\*\*: Gửi yêu cầu trả hàng (lý do, hình ảnh minh chứng) cho các đơn hàng đã nhận, theo dõi trạng thái xử lý (PENDING, APPROVED, REJECTED, COMPLETED). - \*\*Theo dõi đơn hàng:\*\* Xem trạng thái (Chờ xác nhận / Đang xử lý / Đang giao / Đã giao / Đã huỷ) - \*\*Lịch sử đơn hàng:\*\* Xem danh sách và chi tiết các đơn đã đặt - \*\*Đánh giá sản phẩm:\*\* Gửi đánh giá (rating + comment) cho sản phẩm đã mua #### B. Chức Năng Quản Trị Viên (Admin) - \*\*Quản lý danh mục:\*\* Thêm / sửa / xoá / tìm kiếm danh mục, phân cấp danh mục cha-con - \*\*Quản lý sản phẩm:\*\* Thêm / sửa / xoá / tìm kiếm sản phẩm; quản lý biến thể (màu sắc, kích thước, giá, tồn kho), upload ảnh sản phẩm - \*\*Quản lý đơn hàng:\*\* Xem danh sách đơn hàng, tìm kiếm / lọc theo trạng thái, cập nhật trạng thái đơn, xem chi tiết đơn hàng - \*\*Quản lý trả hàng:\*\* Nhận và xem xét yêu cầu trả hàng từ khách, cập nhật trạng thái giải quyết, xác nhận đã nhận lại hàng và thực hiện ghi nhận hoàn tiền. - \*\*Quản lý Bán hàng tại quầy:\*\* Tra cứu nhanh, tạo đơn hàng nhanh, thông tin khách hàng, thanh toán offline, in hóa đơn - \*\*Quản lý người dùng:\*\* Xem danh sách khách hàng, khoá / mở khoá tài khoản, xem lịch sử mua hàng của từng khách - \*\*Quản lý khuyến mãi:\*\* Tạo / sửa / xoá voucher giảm giá (theo % hoặc số tiền cố định), đặt thời hạn và giới hạn sử dụng - \*\*Báo cáo & Thống kê:\*\* Doanh thu theo ngày / tháng / năm, sản phẩm bán chạy, số đơn hàng theo trạng thái ### 6.2 Chức Năng Ngoài Phạm Vi (Out of Scope) - Tích hợp cổng thanh toán thực tế (VNPay, MoMo, Stripe) \*(Planned for future release)\* - Đăng nhập bằng mạng xã hội (Google, Facebook OAuth) - Tích hợp Google Maps để theo dõi vận chuyển - Hệ thống điểm thưởng (Loyalty Points) -- - ## 7.
